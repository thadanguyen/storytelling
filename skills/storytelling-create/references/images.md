# Images Reference

This reference distills Chapter 8 of *Storytelling with You* into creation guidance for images, photographs, illustrations, diagrams, and maps.

## Core principle

Images should serve audience cognition, not slide aesthetics.

The practical implication:

- do not add images because a slide feels empty,
- choose images that help the audience see, understand, feel, remember, or stay oriented,
- pair every image with the message it should help the audience recall,
- choose the image type based on the job,
- and treat diagrams with the same declutter-and-focus discipline used for words and graphs.

## 1. Decide whether an image belongs

Include an image when it helps the audience:

- see a concrete thing,
- understand a process or relationship,
- remember a concept,
- feel the intended tone,
- trust the presenter or situation,
- stay oriented across sections,
- or picture an option, product, place, person, scenario, or prototype accurately.

Move the image out or skip it when it:

- fills empty space,
- decorates a weak slide,
- repeats a stale stock metaphor,
- distracts from the point,
- creates a tone mismatch,
- or makes the deck feel less credible.

Working question:

> What should the audience understand, feel, remember, or navigate better because this image is here?

## 2. Pair image and message

An image is useful when recalling the image also recalls the point.

For each image, write:

- the message it supports,
- the audience reaction it should create,
- where attention should land first,
- what the presenter will say while it is visible,
- and whether the image should appear once, recur, or build over time.

Avoid memorable images that are disconnected from the actual argument. They may be recalled, but they will not strengthen the story.

## 3. Choose image type by function

| Need | Use | Watch for |
|---|---|---|
| Show the actual thing | Photograph | Generic stock photography or tiny corner photos |
| Show something that does not exist yet | Illustration | Style that makes the topic feel unserious |
| Clarify a process, path, system, or relationship | Diagram | Too many boxes, arrows, borders, or equal-emphasis elements |
| Show geographic meaning | Map | Maps used when geography is incidental |
| Orient the audience across sections | Navigation image or diagram | Repetition without a clear current-location signal |
| Set tone before a sensitive ask | Photograph or illustration | Emotional image that feels manipulative or self-indulgent |

Do not choose the visual form because it looks impressive. Choose it because it reduces the audience's work.

## 4. Use photographs deliberately

Use photographs for concrete people, places, products, scenes, evidence, or settings.

Practical rules:

- commit to important photos by making them central,
- crop rather than stretch,
- preserve aspect ratio,
- use high-resolution images,
- avoid stale metaphors like handshakes, bullseyes, money piles, and globes,
- prefer situation-specific photos over generic stock images,
- and read licensing or attribution requirements.

Use the Rule of Thirds when composing or cropping:

- divide the frame into a three-by-three grid,
- place the key subject near a line or intersection,
- and leave natural open space when text must sit on the image.

## 5. Use illustrations for tone and specificity

Illustrations can make material feel:

- approachable,
- exploratory,
- playful,
- informal,
- conceptual,
- simplified,
- or future-facing.

Use illustrations when:

- the object does not exist yet,
- the real thing is visually messy,
- you need control over exact details,
- you want to lower the formality of the material,
- or the audience should feel invited into a working mode.

Avoid clip art when credibility matters. Avoid comics unless the audience, topic, and point justify the tonal risk.

## 6. Use diagrams for structure

Use diagrams when the audience needs to see:

- sequence,
- hierarchy,
- relationship,
- process,
- system boundaries,
- causality,
- dependency,
- or a non-linear path.

Start on paper before slideware when the diagram is conceptually important.

Paper helps test:

- shape,
- flow,
- grouping,
- emphasis,
- whether the structure matches the idea,
- and whether the slideware template is forcing the wrong model.

## 7. Declutter and focus diagrams

Common diagram clutter:

- heavy borders,
- decorative shapes,
- excessive arrows,
- too much color,
- redundant labels,
- equal emphasis everywhere,
- and detail that belongs in speaker notes.

Better defaults:

- push supporting structure into gray or the background,
- reserve color for the current node, step, or relationship,
- remove borders unless they clarify grouping,
- use simple labels,
- and let the presenter explain detail verbally.

Use progressive reveal when the audience should not jump ahead.

Use overview-detail-return when the full structure matters:

1. Show the simplified whole.
2. Emphasize the current section.
3. Move into detail.
4. Return to the simplified whole before the next detail.

## 8. Use images differently by delivery mode

Live presentation:

- blank slides can return attention to the presenter,
- image reveals can create strong moments,
- and a recurring image can support physical-room orientation.

Virtual presentation:

- blank slides can look like technical failure,
- attention is easier to lose,
- simple contextual images can give the eyes something low-effort to rest on,
- and progressive reveals can help pacing.

Standalone document:

- images need more surrounding explanation,
- captions and labels matter more,
- and speaker-dependent reveals may need to become visible structure.

## 9. Use images in the story flow

Images can appear as:

- setup before evidence,
- proof of a concrete situation,
- navigation between sections,
- visual memory hook after an explanation,
- emotional tone-setter before a difficult ask,
- or entity marker reused across data slides.

For repeated images, keep the image stable and vary only the emphasis. This helps the audience recognize where they are without relearning the visual.

## Creation checklist

- [ ] Each image has a written communication job.
- [ ] The image supports a specific message, not a generic topic.
- [ ] The image type fits the audience need.
- [ ] Photographs are central when they matter.
- [ ] Images are cropped, not stretched.
- [ ] Licensing and attribution are handled.
- [ ] Illustrations match the desired tone.
- [ ] Diagrams clarify instead of displaying complexity.
- [ ] Diagram clutter has been removed or muted.
- [ ] Emphasis tells the audience where to look first.
- [ ] Repeated images support navigation or memory.
- [ ] Delivery mode changes have been considered.
