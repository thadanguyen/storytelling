# Graphs Reference

This reference distills Chapter 7 of *Storytelling with You* into creation guidance for data slides and graph sequences.

## Core principle

Data should serve the story. A graph belongs in the deck only when it helps the audience see a specific point, understand an implication, or move toward the desired action.

The practical implication:

- do not include data just because it exists,
- articulate the graph takeaway before choosing the graph form,
- choose familiar graph forms unless novelty earns its cost,
- declutter to reduce effort,
- focus attention to create meaning,
- and weave graphs into the story with words, slide titles, transitions, and progressive builds.

## 1. Decide whether the data belongs

Include data when it helps:

- inform,
- build credibility,
- support a key idea,
- challenge a misconception,
- make something complex visible,
- frame a decision,
- or drive action.

Move data to appendix or backup material when it:

- proves the analysis but slows the story,
- answers a likely skeptical question,
- is useful only to a subset of the audience,
- contains method, caveats, or sensitivity checks,
- or exists mainly because the team worked hard on it.

Working question:

> What will this data help the audience understand, decide, discuss, or do?

## 2. Articulate the graph takeaway

Before designing the graph, write the point in words.

Use this exercise:

1. Look at the data or rough graph.
2. Write several possible observations.
3. Choose the observation that matters most for this audience and story moment.
4. Turn it into a concise takeaway sentence.
5. Say it out loud.
6. Edit until it answers "so what?"

Use the sentence to guide:

- the slide title,
- graph title,
- chart form,
- labels,
- emphasis,
- and spoken setup.

If there are multiple important takeaways, use multiple graph builds, multiple slides, or a main-story / appendix split.

## 3. Choose form by function

Do not start with "what chart looks good?" Start with the job the graph must do.

| Function | Useful forms | Watch for |
|---|---|---|
| Show a trend over time | Line graph, slopegraph, area chart | Do not over-label every point |
| Compare categories | Bar chart, stacked/diverging bar, dot plot | Sort, group, or highlight to reduce scanning |
| Express a relationship | Scatterplot, waterfall chart, pie chart | Make the interpretation cost worth it |
| Show buildup or contribution | Waterfall, stacked bar, stepwise reveal | Keep the sequence easy to follow |
| Show one key number | Big number, sparse text, simple callout | Add only the context needed for meaning |

Prefer line graphs and bar charts as the default for business presentations. Familiar forms reduce decoding burden.

Use less common forms only when:

- the familiar form hides the point,
- the audience already knows the form,
- the presenter can teach the form quickly,
- or the added interpretive cost produces a clearer story.

## 4. Use words inside graph design

Words should help the audience know where to look and what to see.

Use:

- takeaway titles instead of metric labels,
- axis titles when the plotted values may not be obvious,
- direct labels instead of legends when possible,
- annotations for the "why this matters" moment,
- and short phrases that connect the graph to the surrounding story.

Avoid:

- generic titles like "Customer satisfaction",
- legends that force back-and-forth decoding,
- unexplained units,
- dense notes on the slide surface,
- and graph labels written for the analyst rather than the audience.

## 5. Declutter, then focus

Decluttering removes unnecessary work. Focus tells the audience what matters.

Question these elements:

- graph borders,
- heavy gridlines,
- axis lines and tick marks,
- trailing zeros,
- markers on every data point,
- data labels on every value,
- diagonal text,
- center-aligned labels,
- unnecessary color,
- and default chart decoration.

After decluttering, add focus through:

- color,
- intensity,
- thickness,
- size,
- direct labels,
- annotations,
- sparse markers,
- or animation.

Use contrast sparingly. If everything is emphasized, nothing is emphasized.

## 6. Weave graphs into the story

Most business data slides work best in the middle of the story:

- context comes first,
- the graph creates understanding,
- then the presentation moves toward implication, recommendation, decision, or discussion.

For each graph, decide:

- what the audience needs to know before seeing it,
- whether a setup slide is needed,
- how the title prepares interpretation,
- what the presenter should say before the data appears,
- what should appear first,
- what should be revealed later,
- and how the graph connects to the next slide.

Do not make the audience decode the graph while the presenter keeps talking. Either make the graph immediately clear, build it step by step, or pause when reading/inspection is required.

## 7. Build complex graphs step by step

Use progressive builds when the graph is dense, unfamiliar, sequential, or important enough to deserve guided interpretation.

Possible build sequence:

1. Set the stage with a sparse title or context slide.
2. Introduce the axis, scale, or baseline.
3. Add one data point, category, or series.
4. Explain that piece.
5. Add the next piece.
6. Continue until the complete graph appears.
7. Land the takeaway and connect it to the next story step.

This is especially useful for virtual presentations, where attention is fragile and motion can keep the audience oriented.

## 8. Use appendix support for data detail

Use the appendix for:

- raw tables,
- full analysis outputs,
- method notes,
- assumptions,
- caveats,
- sensitivity checks,
- alternate cuts,
- and supporting evidence for skeptical questions.

The appendix should be navigable. Organize it around likely questions, not around the analyst's file structure.

## Creation checklist

- [ ] Each data slide has a reason to exist.
- [ ] Each graph has a written takeaway sentence.
- [ ] The graph form follows the function.
- [ ] Familiar forms are used unless novelty earns its cost.
- [ ] Words clarify what to see.
- [ ] Clutter has been removed or muted.
- [ ] Focus makes the intended point visually dominant.
- [ ] Dense or unfamiliar graphs are built progressively.
- [ ] Data slides connect to the surrounding story.
- [ ] Raw tables and backup analysis are in appendix unless they are the point.
