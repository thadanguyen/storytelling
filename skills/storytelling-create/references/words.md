# Words Reference

This reference distills Chapter 6 of *Storytelling with You* into creation guidance for slide wording.

## Core principle

Words on slides are design material. They should guide audience attention, not merely preserve what the presenter wants to say.

The practical implication:

- use titles to foreshadow and state takeaways,
- put speaker memory support in notes, not on the slide,
- make text-heavy slides scannable or intentionally paced,
- repeat the Big Idea only after context has changed,
- and use closing words to focus memory, discussion, or action.

## 1. Presentation titles should foreshadow

A descriptive title names the topic. A stronger title hints at the point, tension, or decision.

Weak:

- "Supplier Analysis"
- "Competitive Landscape Update"
- "Quarterly Business Review"

Stronger:

- "Changing our supplier strategy could reduce costs"
- "We are winning across the competitive landscape"
- "(Mostly) strong results last quarter"

If a pointed title feels too abrupt, combine a descriptive title with a foreshadowing subtitle.

## 2. Slide titles should carry takeaways

Use slide titles as audience-facing cues.

Working rule:

- identify the slide's main point,
- write it as a sentence,
- shorten it until it fits cleanly,
- and keep it connected to the Big Idea and story path.

Use action verbs when the slide should prompt behavior: understand, discuss, decide, compare, choose.

## 3. Text slides need an attention contract

Before writing slide text, decide what the audience should do with it:

- **Read:** pause and let them read.
- **Scan:** use hierarchy so the gist is visible quickly.
- **Listen:** keep the slide sparse and let the speaker carry the explanation.
- **Discuss:** show the question, recommendation, or decision prompt.
- **Remember:** use a stable phrase or Big Idea callback.

Do not talk over text that the audience must read.

## 4. Separate speaker memory from audience attention

Teleprompter slides confuse speaker support with audience support.

Practical test:

- If the text is there because the presenter needs it, move it to speaker notes, a rehearsal outline, or a cue card.
- If the text is there because the audience needs it now, it can belong on the slide.

For bullet-heavy slides:

1. Identify the main point of each bullet.
2. Rewrite each point as a short phrase.
3. Keep only the audience-facing cue on the slide.
4. Move the explanation into speaker notes.
5. Reveal points gradually if reading ahead would hurt delivery.

## 5. Make more text scannable when it must stay visible

Text-heavy slides are not automatically wrong. They fail when they force reading while the presenter is speaking.

Use hierarchy:

- large text for the main idea,
- bold for key words,
- sparing color for the most important phrase,
- gray or smaller type for secondary context,
- and placement that separates main point from support.

## 6. Handle mid-story text deliberately

When text appears in the middle of the story, choose one of three treatments:

| Treatment | Use when | Presenter behavior |
|---|---|---|
| Pause to read | The audience needs the whole quote or passage | Introduce it, stop talking, then make the point |
| Reveal piece by piece | The order matters or virtual delivery needs pacing | Build text gradually as you explain |
| Selectively highlight | Only part of a longer text matters | Make the important words visually dominant |

## 7. Repeat the Big Idea only when it gains meaning

A Big Idea should not return as a flat slogan. It should return after the audience has gained new context.

Use this progression:

| Moment | Function |
|---|---|
| Beginning | Frames the opportunity or destination |
| Middle | Lets evidence, examples, tension, and stakes attach to one idea |
| End | Compresses the argument into a usable decision frame |
| Afterward | Gives the audience language they can repeat |

Constraints:

- stable wording, so the phrase is recognizable,
- expanding meaning, so each return reveals more inside the phrase,
- earned repetition, with evidence or stakes between returns,
- real compression, not just a catchy line,
- audience usability, so people can actually say it later.

## 8. Placement changes what the same sentence does

The same sentence can shift from frame to conclusion.

- Before evidence, it points forward: "watch for this."
- After evidence, it gathers backward: "this is what the path has shown."

Design wording and placement together. If trust is low, an early Big Idea may feel like an unsupported claim; if the Big Idea appears only at the end, it may feel abrupt.

## Creation checklist

- [ ] The presentation title foreshadows the point or tension.
- [ ] Slide titles state takeaways, not topics.
- [ ] Each text slide has a clear attention contract.
- [ ] Speaker-memory text is in notes, not on the slide.
- [ ] Dense text is scannable, paced, or given reading time.
- [ ] Big Idea callbacks are earned by new context.
- [ ] Closing words direct memory, discussion, or action.
